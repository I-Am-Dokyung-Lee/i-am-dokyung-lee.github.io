Problem Solving

멀티턴 대화 맥락 유지
상황 단일 질문-응답만 가능하여 실제 경영 상담처럼 연속적인 대화가 불가능
해결 Redis 세션 메모리 + 대화 히스토리 요약 파이프라인 설계. TTL 정책으로 메모리 관리
결과 이전 질문 참조 심화 상담 가능, 실제 컨설팅에 가까운 UX 구현
RAG 파이프라인 성능 고도화
상황 Faithfulness 0.36, Context F1 0.35로 답변 신뢰성 부족
해결 5단계 반복 개선 — 청킹 전략, 한국어 프롬프트 최적화, Cross-Encoder 재순위화, RAGAS 배치 평가 CLI
결과 Faithfulness +72.5%, Context F1 +79.7%, 거부 정확도 0% → 100%
