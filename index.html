문제 해결 사례

멀티턴 대화 맥락 유지

상황: 단일 질문-응답만 가능하여 실제 경영 상담처럼 연속적인 대화가 불가능했음

해결: Redis 세션 메모리 + 대화 히스토리 요약 파이프라인 설계. TTL 정책으로 메모리 관리, 토큰 제한 내에서 맥락 유지

결과: 사용자가 이전 질문을 참조하며 심화 상담이 가능해져 실제 컨설팅에 가까운 UX 구현

RAG 파이프라인 성능 고도화

상황: 초기 RAG 시스템의 Faithfulness 0.36, Context F1 0.35로 답변 신뢰성 부족

해결: 5단계 반복 개선 — 청킹 전략 변경, 한국어 프롬프트 최적화, Cross-Encoder 재순위화 적용, RAGAS 배치 평가 CLI 구축으로 정량적 개선 사이클 확립

결과: Faithfulness +72.5%, Context F1 +79.7% 달성. 거부 정확도 0% → 100% 완전 달성

기술 선택 근거

  • LangGraph: 4개 도메인 분류 → 검색 → 생성 → 평가의 복잡한 분기 로직을 상태 머신으로 관리. LangChain의 선형 체인 대비 조건부 분기, 루프, 병렬 처리 지원
  • ChromaDB: 로컬 환경에서 빠른 프로토타이핑, 청크 메타데이터 필터링 용이. Pinecone 대비 비용 절감, FAISS 대비 메타데이터 관리 편의성
  • Redis: 세션 기반 대화 맥락 저장에 최적. TTL 자동 만료로 메모리 관리, 인메모리 DB로 빠른 읽기/쓰기
